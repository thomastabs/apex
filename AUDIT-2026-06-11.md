## Executive Summary

Codebase is in good shape: thin routers, clean service layer, real adapter abstraction,
meaningful prompt engineering. But the audit found **one shipped regression that breaks
Phase 4**, **two systemic security gaps** (no real authentication; SSRF bypass via the
config path), and a **re-introduced NaN bug** in the Taiga adapter context. None are
caught by the current test suite вҖ” Phase 4 has zero backend tests, and the latest
refactor (`6147855`) deleted functions that production code still calls.

## 1. Functionality & Bug Discovery

### рҹ”ҙ C1 вҖ” Phase 4 `story-context` endpoint broken (regression from commit `6147855`)
- [x] **FIXED 2026-06-11**

- Fix: drop `task_list` from `Phase4Service.get_story_context` + `StoryContextResponse`; delete dead `ContextService.save_task_list/load_task_list/load_all_task_lists`.

### рҹҹ  H1 вҖ” Taiga slugвҶ’NaN bug re-introduced in Phase 3/4 hooks
- [x] **FIXED 2026-06-11**
- `frontend/lib/hooks/use-phase3.ts:110-116` (`getAdapterCtx`) and `use-phase4.ts:118-123` use `context.pmProjectId ?? String(context.projectId)` unconditionally.
- `project-section.tsx:64` stores `pmProjectSlug` for ALL PM tools; Taiga projects always have slugs вҶ’ `Number(slug)` = `NaN` вҶ’ `/tasks?project=NaN`.
- Same bug class as the session-28 `toPmCtx` fix (`workspace.ts:14-25`); the fix was never propagated to the hooks.
- Fix: tool-aware branching (`pmTool === "jira"` вҶ’ slug; else numeric id), shared between all call sites.

### рҹҹ  H2 вҖ” Story-index cache incoherent across workers, racy within one
- [ ] `context_manager._story_index_caches`: process-global dict, no invalidation, no lock.
- Cross-process: with >1 uvicorn worker, worker A's index write invisible to worker B forever. (Dockerfile currently `--workers 1`; CLAUDE.md claims 2 вҖ” doc drift.)
- In-process: phase endpoints are sync `def` вҶ’ threadpool; `upsert_story_index` is unlocked read-modify-write вҶ’ concurrent gate transitions can lose updates.
- Fix: mtime-based re-read (or drop the cache) + `threading.Lock` around index mutations.

### рҹҹЎ M1 вҖ” Rehydrated task DAG can point at wrong predecessors
- [ ] `useLoadTaskList` (`use-phase3.ts:274-285`) reassigns local ids `1..n` by PM-id order; `predecessor_task_ids` from metadata reference *original* ids. Task deleted in PM вҶ’ all edges after the gap shift.
- Fix: persist original local id inside `encodeApexMeta` block; use it on decode.

### рҹҹЎ M2 вҖ” `decodeApexMeta` eats user content containing `---`
- [ ] `APEX_META_BLOCK_RE` (`use-phase3.ts:35`) matches from the FIRST `\n\n---\n\n` to end. Description containing a markdown hr loses content on decodeвҶ’encode round trip.
- Fix: anchor to last block or use unambiguous marker comment.

### рҹҹЎ M3 вҖ” Body-size middleware: docstring wrong, chunked path broken
- [ ] `main.py:48-76`: `@app.middleware("http")` IS BaseHTTPMiddleware (docstring claims otherwise). No-Content-Length branch assigns `request._stream` вҖ” not a Starlette attribute; body lost downstream for chunked requests.
- Fix: reject chunked with 411, or set `request._body`.

### рҹҹЎ M4 вҖ” `phase4_passed` stat silently stripped
- [ ] `workspace.py:190` returns `phase4_passed` but `StoryIndexStatsResponse` doesn't declare it вҶ’ response_model drops it. Nothing consumes it. Add field or remove computation.

### рҹ”ө L1 вҖ” `_invoke` assumes `response.content` is str (`ai_engine.py:294`); Anthropic can return content-block lists (fallback path already handles it). Extract shared normalization.
### рҹ”ө L2 вҖ” `persist_design` (`phase2_service.py:120`) writes endpoints *markdown* into a ` ```yaml ` block via `write_project_technical_spec`. Mislabeled content for downstream prompts.

## 2. Security & Authentication

> Assumes backend is internet-reachable (Azure Container Apps public ingress per ci.yml).

### рҹ”ҙ C2 вҖ” No actual authentication on the backend
- [ ] `deps.py:get_auth_context` accepts ANY non-empty Bearer string; never validated. `X-Project-Id` trusted blindly. Anonymous client can: read every project's context files, overwrite/wipe them, rewrite global workspace config, burn AI budget.
- Rate limiter keys on `sha256(token)` of the attacker-chosen token вҶ’ rotate token to bypass the 20 req/min cap.
- Fix: validate token against the configured PM tool (`GET /users/me`, cached ~60s by token hash) + verify project membership for `X-Project-Id`. Add a global per-IP cap.
- **Product decision**: if Apex stays single-user demo, may consciously accept + document as known limitation in thesis. C3 must be fixed regardless.

### рҹ”ҙ C3 вҖ” SSRF: config-path URLs bypass all validation
- [ ] `taiga_proxy.py:128-138` (`_get_taiga_url` config branch): no https/private-host check.
- [ ] `jira_proxy.py:58-73` (`_get_jira_base_url`): no validation at all; `*.atlassian.net` restriction applies only to the header override.
- [ ] `SaveConfigRequest.jira_base_url` has only `max_length=2048`; `save_config` requires only the fake bearer auth.
- Attack: `POST /workspace/config {"pm_tool":"jira","jira_base_url":"http://169.254.169.254"}` вҶ’ any `/api/pm/jira/*` call hits cloud metadata service, response returned verbatim.
- Fix: run the same validators on the config path in both proxies AND at save time in `save_config`.

### рҹҹ  H3 вҖ” DNS-rebinding hole in SSRF guard
- [ ] `_is_blocked_host` (both proxies) only blocks IP literals + "localhost". Public hostname resolving to private IP passes.
- Fix: resolve hostname; check every resolved address against `_BLOCKED_NETS`; pin resolved IP for the request.

### рҹҹЎ M5 вҖ” Open auth-relay on `/api/pm/taiga/auth`
- [ ] Unauthenticated, un-rate-limited, forwards credentials to any public https host's `/api/v1/auth` вҶ’ credential-stuffing relay hiding attacker IP. Add IP rate limit; consider host allowlist.

### рҹҹЎ M6 вҖ” Prompt injection surface
- [ ] User text (epic descriptions, hints, QA notes) and **`github-context.md`** (README from an EXTERNAL repo) injected into prompts undelimited. Hostile README вҶ’ injected into Phase 2/3 generation вҶ’ persisted into specs feeding later phases (cross-phase chain).
- Fix: fence untrusted segments with "data, not instructions" preamble; cap github-context length; keep DOMPurify invariant on all markdown render surfaces.

### рҹҹЎ M7 вҖ” XSS blast radius: tokens in sessionStorage + permissive CSP
- [ ] CSP has `'unsafe-inline' 'unsafe-eval'`, `connect-src вҖҰ https:` (any host). Single XSS exfiltrates PM tokens anywhere. Long-term: nonce-based CSP via middleware; tighten connect-src to API + GitHub.

### рҹҹЎ M8 вҖ” Proxy responses unbounded
- [ ] 4 MB cap is request-only; `resp.content` from Taiga/Jira buffered fully with no limit. Add response size cap (~10 MB) or stream.

**Done well:** SSRF netblock list for literals; `\r\n` header-injection guards; token length caps; PAT excluded from persistence; `save_config` pops `auth_token`; `follow_redirects=False` on proxy clients.

## 3. Architecture & SDLC Alignment

### рҹҹ  H4 вҖ” Workspace config is a global singleton (accidental single-tenancy)
- [ ] `.apex-config.json` holds `pm_tool`/`jira_base_url`/`project_id`/`ai_model` for the entire backend. `set_active_project` (`context_manager.py:158-168`) WRITES `project_id` on every project switch вҖ” write in a read path, thrashes under concurrent users; user A switching to Jira breaks user B's Taiga config-path proxy.
- Fix: scope config per project dir (or per user); remove the `save_config` side effect from `set_active_project`.

### рҹҹЎ M9 вҖ” Phase boundary leaks (otherwise clean)
- [ ] `phase4_service` imports `context_manager` directly (lines 101, 139, 152) instead of `ContextService` вҖ” only service that does.
- [ ] `workspace.py:86`: `ai_engine._llm_cache.clear()` вҖ” router touching a private; clears cache during in-flight AI calls on other threads.

### рҹҹЎ M10 вҖ” AI engine blocking retries multiply
- [ ] `_ai_retry` (10s + 30s sleeps) wraps clients already at `max_retries=2`, on sync endpoints on the ~40-thread anyio pool. Burst of 429s can starve ALL sync endpoints.
- Fix: async AI endpoints with `asyncio.sleep`, or cap combined retry budget, or dedicated smaller threadpool.

### рҹ”ө L3 вҖ” `get_model()` re-reads config from storage per AI call (network round trip in Azure mode). Cache with short TTL.
### рҹ”ө L4 вҖ” `context_manager.__getattr__` magic masks deletions (why C1 surfaced at request time, not import time). Plan removal; tests are the only consumer.
### рҹ”ө L5 вҖ” `PHASE_STATUSES` lacks `new` status that CLAUDE.md documents. Align.

## 6. Phase 5 вҖ” Research & Planning

`ai_engine.py:1460-1467` TODO + placeholder page frame Phase 5 as **Deployment**.

### Feature A вҖ” Release Readiness & Deployment Gate (natural Phase 5)
- Release manifest from `qa_passed` stories; AI release notes from Gherkin + bug reports.
- `generate_infra_delta` (existing TODO): DevOps persona scans tech spec вҶ’ `INFRA_DELTA: NONE | REQUIRED <delta>`.
- Deploy checklist + human gate вҶ’ `mark_story_deployed` (already implemented) вҶ’ PM status transition.
- Effort: low вҖ” reuses every existing pattern (1 prompt, ~3 endpoints, 1 workflow component).

### Feature B вҖ” Code Verification Loop (highest thesis value)
- Fetch merged PR diff via existing GitHub client; AI-as-judge: diff + Gherkin + pack test assertions вҶ’ **traceability matrix** (scenario вҶ’ evidence вҶ’ covered/uncovered/contradicted).
- Closes the feedback loop: Apex currently generates instructions but never verifies results. Pre-seeds Phase 4 QA with "scenarios the diff doesn't cover."

### Feature C вҖ” SDLC Analytics & Retrospective (evaluation instrument)
- Add timestamps to story-index transitions; dashboard: cycle time per phase, Fix-Bolt rate per epic, vaccine recurrence, AI token/cost per story (log in `_invoke`).
- AI sprint retrospective grounded in the data. Gives quantitative thesis evaluation data.

**Recommendation:** A + B for Phase 5; C alongside as the evaluation instrument. Skip full agentic autonomy (auto-executing packs) вҖ” thesis-sized scope expansion; the human gate IS the framework's point.
