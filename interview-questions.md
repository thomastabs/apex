# Apex — Tech Lead Interview Questions

Apex = spec-driven SDLC framework. Phases 1–6, traceability, PM integration (Taiga/Jira), Figma grounding, multi-model cross-check, Azure deploy.

Goal: open-ended, surface real critique, not yes/no. Lead with demo, end with "what would stop you adopting."

---

## Filipe — Oktogon — DevOps Tech Lead

*Angle: spec-driven flow + integrations — cut real delivery pain or add ceremony.*

### Open — the core pain
1. "Apex locks Gherkin (P1) → design (P2) → tasks/dev-packs (P3) before any code lands. In your delivery cycles, where does drift between intent and shipped code actually cost you — and would these gates catch it?"
2. "Phase status is a hard gate (gherkin_locked → … → deployed). Does enforced sequencing help teams, or do good teams route around it?"
3. "Spec is the source of truth, code follows. When reality inverts — hotfix ships first, spec catches up after — does the model break, or bend?"

### Phase specifics
4. "Phase 3 deploy packs — file-level coding proposals tied to scenarios, like Claude Code plans. Survive contact with your CI, or get regenerated by hand?"
5. "Phase 5 verification matrix (scenario→test coverage) + analytics. The trace evidence your release reviews want, or audit theater?"
6. "Phase 4 Fix Bolt: bug → fix log → patch scope, tied to the failing scenario. Better incident hygiene, or slows a 2am hotfix?"
7. "Phase 6 ingests Jira issues + Figma comments as maintenance triage. Does closing the loop back to spec change how you handle prod incidents?"
8. "Phase 1 reconstructs Gherkin from an existing Taiga project on import. Realistic retrofit onto a live project, or greenfield-only value?"
9. "Phase 2 preset tech stacks seed the design draft. In an opinionated org, does a guided stack picker help onboarding or fight your golden path?"

### Integrations
10. "Code is verified against the Phase 3 dev pack. How much would you trust spec-conformance vs. your existing PR review?"
11. "Where in your GitHub flow (branch, PR, CI gate, merge) does spec-traceability add value vs. noise in the diff?"
12. "Phase 1 frames→stories, Phase 2 frames→screen flow. Does design-as-source-of-truth match how your designers and devs actually hand off?"

### Autopilot + ops reality
13. "Autopilot runs P1–5 unattended with checkpoints. Where do you want a hard gate autopilot never crosses — does Apex put it there?"
14. "AI rate-limits or times out mid-phase, run fails part-way. Acceptable failure: resume, roll back, or hard-stop with a clear artifact?"
15. "Deploy packs are AI-generated plans. Your minimum bar before one auto-opens a PR vs. stays a human-reviewed proposal?"

### Adoption — close
16. "Everything's keyed to a PM tool (Taiga/Jira) as the spine. If your truth is GitHub Issues or Linear, still adoptable or DOA?"
17. "What integration cost — auth, secrets, CI wiring — kills this regardless of how good the methodology is?"
18. "Single metric that would prove Apex cut delivery time, not just added ceremony?"

## Ricardo Amaro — Spec-Driven / AI-Dev Tech Lead

*Angle: is the methodology sound — does phase-gating + grounding raise AI output quality.*

### Open — is this real
1. "SDD isn't new — BDD, Cucumber, formal methods tried it and mostly died on maintenance cost. What makes AI-era spec-driven survive where human-maintained specs rotted?"
2. "How does Apex compare to what you've seen — spec-kit, Kiro, plain Cursor/Claude with a rules file? Is phase-gating a real differentiator or repackaging?"
3. "Core bet: a locked spec makes AI codegen more reliable than prompt-and-pray. Is spec rigor actually the bottleneck on AI output quality — or is it context, model, eval?"
4. "Apex makes the spec the durable artifact, treats code as regenerable from it. Buy spec-as-source-of-truth, or is code still the only truth that matters?"

### Spec rigor
5. "Apex locks Gherkin AC before design before code. Real win, or front-loading work the spec changes out from under anyway?"
6. "Who writes the spec — human authors Gherkin and AI implements, or AI drafts it too? Where's the right human/AI split for the spec to stay trustworthy?"
7. "How do you stop the model agreeing with a bad spec instead of challenging it? Does any phase here do that?"
8. "What's your bar for when AI-generated Gherkin AC is trustworthy enough to lock?"
9. "Phase 1 multi-model cross-check runs two providers, set-diffs scenarios (agreed/only-each). Does cross-model agreement raise requirements quality, or manufacture false confidence?"

### Traceability
10. "Every artifact links scenario→design→endpoint→code→test, both directions. How much of that trace value is real vs. theory?"
11. "Backward-trace: a P5/6 gap re-opens the SOURCE spec phase (scenario→P1, endpoint→P2). Right direction of correction, or should fixes only flow forward?"
12. "Decision log captures rejected options as negative constraints injected into later prompts. Useful steering, or prompt pollution that compounds?"

### Grounding — Figma + code
13. "Phase 3 injects Figma design tokens + linked frame image into the coding proposal. Real signal for codegen, or noise the model ignores?"
14. "Phase 4 grounds the QA test plan in the Figma frames too. Does design-grounded test-gen catch what spec-only misses?"
15. "Drift detection is deliberately pure — Jaccard, hashing, set-diff, no AI. Where would you have put AI in that loop that we left out?"
16. "Phase 3 dev packs; later phases verify code against them. Is conformance-to-proposal the right correctness signal, or too rigid for how code evolves?"
17. "Provider-agnostic by model-ID (Claude/GPT/Gemini). Does multi-provider help quality, or fragment your eval and reproducibility?"

### Provoke — close
18. "Every phase produces a reviewable artifact. Does forcing artifacts at each gate raise quality, or generate documents nobody reads?"
19. "What kind of project does this pay off on — greenfield SaaS, regulated/compliance, large-team coordination — and where is it pure overhead?"
20. "Where's the methodology weakest — which phase is most likely theater that teams skip once novelty wears off?"
21. "If AI codegen gets 10x better in two years and one-shots features from a loose prompt, does a phased spec framework become obsolete, or more necessary?"
22. "What convinces you Apex is methodology, not glued-together tooling — and what fails after 6 months a demo won't show?"

## Closers (both)

- "Would you use this on your team — why / why not?"
- "What would you steal from Apex even if you never adopted the whole thing?"

**Arc logic:** Filipe runs pain → phases → integrations → ops → adoption (concrete to commitment). Ricardo runs is-this-real → rigor → trace → grounding → provoke (frame his stance early, push hardest at the end when he's invested). Both end on the steal/adopt verdict.
